# Project Structure

## 设计目标

这个仓库不是只放零散 `.pine` 文件，而是要变成一个可持续维护的 Pine Script 开发仓库。

核心原则：
- **脚本分类清晰**：指标、策略、库分开
- **研究过程可追踪**：思路、参数、结论有文档沉淀
- **方便复用**：公共函数统一收敛到 libraries
- **便于扩展**：后续加自动化、回测记录、发布清单时不用重构整个仓库

## 分层说明

### 1. docs/

保存项目级文档：
- 结构说明
- 开发约定
- 命名规范
- 协作规则

### 2. pine/indicators/

放纯指标脚本。按方向分类：
- `trend/`：趋势类
- `momentum/`：动量类
- `volatility/`：波动率类
- `volume/`：成交量类

### 3. pine/strategies/

放可回测策略脚本。按策略范式分类：
- `trend-following/`
- `mean-reversion/`
- `breakout/`

### 4. pine/libraries/

放公共逻辑，例如：
- MA 计算
- ATR/波动率工具
- 风险控制辅助
- 信号过滤函数
- 会话/时间窗口工具

### 5. pine/templates/

用于快速起项目：
- 指标模板
- 策略模板
- 库模板

### 6. research/

把研究过程与代码分开管理：
- `ideas/`：想法草稿
- `notes/`：实验记录、参数变化、结论

### 7. tests/

Pine 不像普通编程语言那样容易自动化单元测试，因此这里主要保存：
- 手工验证 checklist
- 回测观察项
- 视觉核验项
- 多周期/多品种测试记录

### 8. examples/

放最小示例：
- 新人入门示例
- 演示公共库如何使用
- 演示一个完整指标或策略的写法

## 建议后续扩展

未来如果项目变大，可以继续增加：
- `releases/`：发布版脚本
- `assets/`：截图、图表、文档图片
- `changelog/`：版本变更记录
- `automation/`：辅助脚本，如格式化、导出、文档生成
